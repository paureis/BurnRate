# BurnRate v5 — Progress Log

Tracks each feature from `docs/specs/goal5.md` through Started → Decisions → Tests added → Verified.

## Feature 1 — Usage tracker & ROI — Wired (display)

- **Started**: 2026-05-14
- **Decisions**: `Subscription.usageLog?: Record<string, UsageEntry>` keyed by `"YYYY-MM"`. `src/lib/usage.ts` exports `scoreUsage`, `recordUsage`, `shouldNudgeForMonth`, `markNudgeShown`. Badge rules: **hero** when ≥3 used and cost-per-use ≤ $1.00; **zombie** when ≥1 used and the trailing zero-use run hits 3+; **ghost** at zero use and 2+ months tracked; otherwise **steady/mixed/untracked** based on usage ratio.
- **UI**: `UsageInsights.tsx` dashboard module renders top-3 heroes, zombies, and ghosts with ROI detail strings. Wired into the dashboard between Trends and Ledger. The monthly check-in banner UI is deferred — the data shape and `shouldNudgeForMonth` hook are ready.
- **Tests added**: 13 cases (each badge rule, contiguous zero-use definition, null cost-per-use, immutable `recordUsage`, nudge gating).
- **Verified**: Lib + tests + UI display green.

## Feature 2 — Price-change scheduler — Wired

- **Started**: 2026-05-14
- **Decisions**: `Subscription.priceChanges?: PlannedPriceChange[]`. `src/lib/price-changes.ts` exports `applyDuePriceChanges` (pure + idempotent), `projectMonthlyBurnWithChanges`, `expandPriceChangeTimeline`, `validatePriceChange`, `buildPriceChange`. Boot-time sweep applies changes in date order and removes them from the queue.
- **UI**: `PriceChangeEditor.tsx` is embedded in the subscription add/edit form. Each change shows effective date + new cost + optional note with a delete button; the add row validates via `validatePriceChange`. `BurnRateApp.tsx` boot effect runs `applyDuePriceChanges` after the v3 cancellation sweep and emits a toast when changes apply.
- **Tests added**: 12 cases (no-op shortcut, single + multiple due, date-order application, idempotency, future-date preservation, forecast bend, validate failure modes).
- **Verified**: Lib + tests + integration green.

## Feature 3 — Charge calendar heatmap — Wired

- **Started**: 2026-05-14
- **Decisions**: `src/lib/charge-calendar.ts` exports `buildChargeCalendar` (walks each sub's `nextBillingDate` backward by its `billingCycle` over `horizonDays`, currency-converted) and `summarizeChargeCalendar` (peak day, day-of-month / day-of-week dominance at ≥40%). End-of-month wrap clamps via `stepCycle`. Ledger-cancelled subs skip charges on/after their cancellation date.
- **UI**: `ChargeCalendarPanel.tsx` renders a GitHub-style 7-row grid with intensity steps from the dominant theme color. Tooltips on each cell expose the day's total + charge count. Range toggle between "Last 90 days" and "Last 12 months". Summary cells (active days, total, avg, peak) below the grid, plus the dominantDay-of-month / day-of-week prose when the threshold fires.
- **Tests added**: 11 cases.
- **Verified**: Lib + tests + integration green.

## Feature 4 — Retention/discount log — Wired (display)

- **Started**: 2026-05-14

## Feature 5 — End-of-year report — Wired
