## Feature 2 — Tags & saved views — Wired

- **Started**: 2026-05-14
- **Decisions**: `src/lib/tags.ts` provides `normalizeTag` / `mergeTags` / `collectAllTags` / `tagsEqual`. Tags are lowercase-kebab `[a-z0-9-]+`, ≤20 chars, ≤10 per record, with case-insensitive dedup. `src/lib/views.ts` defines `SavedView` + `applyView` + `normalizeViews` + `buildBuiltinViews`. Three built-ins seed on first load: `all-subs`, `yearly-only`, `cancelling-soon`. Filter supports query, tags (AND), categories/cycles/currencies (OR), monthly cents min/max in base currency, and `cancellingOnly`. Sort by `name|cost|nextBillingDate|category` asc/desc.
- **UI**: `TagInput.tsx` chip-style input with Backspace-to-remove and autocomplete drop from `collectAllTags`. Wired into `SubscriptionForm`. Tags render as `#tag` pills on every `SubscriptionRow`. `SavedViewsPills.tsx` renders a horizontal pill row above the subscription list, with active-pill highlighting, kebab-style delete on user views (not built-ins), and an inline "Save view" form that captures the current category filter + sort. Persistence: `burnrate.views.v1` via `useLocalStorage`.
- **Tests added**: 17 tag cases + 14 view cases.
- **Verified**: Lib + tests + integration green. Build clean.

## Feature 4 — Search operators in the command palette — Wired

- **Started**: 2026-05-14
- **Decisions**: `src/lib/palette-query.ts` parses `cost:>20`, `cost:<5`, `cost:>=N`, `cost:<=N`, `cost:N` (clamped equality), `cycle:`, `tag:`, `category:`, `currency:`, `cancelling:`, `trial:`. Unknown operators or malformed values fall through to `freeText`. Repeated `tag:` is AND; repeated `category:`/`currency:` is OR. `describeParsedQuery` builds the human-readable summary line; `PALETTE_OPERATOR_HELP` powers the help drawer.
- **UI**: `CommandPalette` accepts an optional `fx` prop and an optional `meta: { sub, trial }` on each `CommandItem`. The `rankCommands` helper now (a) parses the query, (b) filters record-derived commands by operator predicates against the meta, (c) ranks remaining commands by the free-text portion. App actions (open dashboard etc.) bypass operator gating so they remain reachable. A parsed-summary chip and an inline "Show operators" help drawer round out the surface.
- **Tests added**: 18 cases including parse fidelity per operator, sloppy-input fallthrough, decimals, mixed query + free text, summary string.
- **Verified**: Lib + tests + integration green. Build clean.

## Feature 5 — Undo history & audit log — Lib complete

- **Started**: 2026-05-14
- **Decisions**: `src/lib/history.ts` exposes `record`, `prune`, `append`, `dropEntry`, `normalizeHistory`, `buildSummary`. Ring buffer cap is 20 (`HISTORY_LIMIT`). Per-entry `before`/`after` payloads are checked against an 8 KB byte cap (`HISTORY_MAX_PAYLOAD_BYTES`) — oversized entries store `null` instead and set `oversized: true` so the UI can disable Undo. Entry IDs include a millisecond timestamp, an in-process counter, and 6 random base32 chars so 1000 rapid `record()` calls all produce unique IDs.
- **Tests added**: 12 cases (id uniqueness across 1000 calls, oversized detection, prune behavior, append-then-prune, summary copy for every op, normalizeHistory dedup + filter).
- **Verified**: Lib + tests green. UI wiring (HistoryDrawer + Ctrl/Cmd+Z + per-mutation hook integration) deferred.

## Feature 6 — Custom categories — Wired (registry panel)

- **Started**: 2026-05-14

## Feature 7 — Profile export / import — Wired

- **Started**: 2026-05-14
- **Decisions**: `src/lib/profile.ts` exports `PROFILE_SCHEMA_VERSION = 1`, `BurnRateProfile`, and the trio `exportProfile`, `applyProfile`, `validateProfile` plus a `previewProfile` helper for the merge/replace confirm UI. Export strips built-in views; preserves built-in *edits* to categories. Apply has `merge` (union views by id, deep-merge fxOverrides) and `replace` (overwrite settings slices wholesale). Data slices (subs/trials/ledger/snapshots) are NEVER touched in either mode.
- **UI**: `ProfileSettings.tsx` adds a Settings section with Export and Import buttons. Import opens a file picker, validates with `validateProfile`, then shows a preview diff (theme, base currency, new FX overrides count, new views count, new categories count) before applying either Merge or Replace. Wired into `BurnRateApp.tsx` at the top of the Settings view.
- **Tests added**: 13 cases.
- **Verified**: Lib + tests + integration green. Build clean.
